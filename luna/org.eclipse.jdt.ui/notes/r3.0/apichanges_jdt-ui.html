jdt ui - Breaking API changes from R2.1 to R3.0
java development tooling ui
  • org.eclipse.jdt.ui.actions.SelectionDispatchAction: changed visibility of the run and selectionChanged methods from protected to public (see bug for 37846 details). As an effect subclasses reimplementing one of the run or selectionChanged methods will not compile anymore. To fix the compile errors make the corresponding methods in the subclasses public as well.
This document lists the breaking API changes that occurred between R2.1 and R3.0 and how to migrate from the R2.1 API to the R3.0 API.
